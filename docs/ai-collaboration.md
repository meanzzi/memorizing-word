# AI Collaboration & Technical Decision Log

## 1. 프로젝트 개요

영어 암기장 웹앱은 브라우저 환경에서 동작하는 Vanilla JavaScript 기반 프로젝트입니다.
단어 CRUD, 퀴즈 시스템, Web Speech API 기반 발음 기능, 다크 모드 등을 포함하고 있으며, 별도의 서버 없이 localStorage를 활용해 동작합니다.

프로젝트 구현 과정에서는 Gemini CLI를 활용해 초기 구조 설계와 기능 구현 속도를 높였습니다.
다만 AI의 제안을 그대로 사용하는 대신 유지보수성·사용자 경험·성능 관점에서 여러 차례 구조를 수정하며 개발을 진행했습니다.

---

# 2. AI 협업 방식

프로젝트 진행 과정에서는 Gemini CLI와 협업하기 위한 별도의 작업 규칙 문서를 정의했습니다.

* 사용자 가치 먼저 정의
* 구현 방향 설명
* 실제 코드 작성
* 실행 방법 정리

순서로 답변하도록 규칙을 설정했고, 커밋 메시지 규칙과 코드 스타일 원칙도 함께 관리했습니다.

이를 통해 AI를 단순 자동 생성 도구처럼 사용하는 대신 일관된 개발 프로세스를 유지하는 협업 도구로 활용하고자 했습니다.

---

# 3. 이벤트 처리 구조 수정

초기에는 AI가 단어 목록의 수정/삭제 버튼마다 각각 이벤트 리스너를 등록하는 구조를 제안했습니다.

초기 구현 단계에서는 문제가 없어 보였지만 단어 개수가 많아질 경우 이벤트 리스너가 계속 증가하면서 메모리 사용량과 유지보수 복잡도가 커질 수 있다고 판단했습니다.

또한 동적으로 추가되는 요소마다 별도로 이벤트를 관리해야 했기 때문에 구조가 점점 복잡해질 가능성이 있었습니다.

그래서 최종적으로는 이벤트 위임 방식으로 수정했습니다. 부모 컨테이너에 하나의 이벤트 리스너만 등록하고, 이벤트 타겟을 기반으로 동작을 분기하도록 구조를 변경했습니다.

이 과정에서 저는 “기능이 동작하는 것”보다, 데이터가 많아졌을 때도 유지보수가 가능한 구조를 더 중요하게 생각했습니다.

---

# 4. 사용자 입력 데이터 검증

초기 단어 추가 기능에서는 대소문자가 다른 경우 동일 단어가 중복 저장되는 문제가 있었습니다.


* Apple
* apple

이 서로 다른 데이터로 저장될 수 있었습니다.

AI는 단순 문자열 비교를 제안했지만 실제 사용자 입장에서는 두 단어를 동일한 단어로 인식한다고 판단했습니다.

그래서 입력값을 lowercase 처리한 뒤 비교하도록 수정해 데이터 무결성을 개선했습니다.

또한 단순 기능 구현보다 “사용자가 예상하는 동작 방식”에 맞는 UX를 만드는 것을 중요하게 생각했습니다.

---

# 5. 다크 모드 애니메이션 수정

초기 다크모드 구현 과정에서는 body와 header의 transition 속도가 서로 달라 화면 전환 시 시각적으로 어색한 느낌이 발생했습니다.

AI는 단순히 transition 속도를 추가하는 방향을 제안했지만, 실제 사용 경험에서는 UI 전체의 일관성이 더 중요하다고 판단했습니다.

그래서 주요 컴포넌트의 transition duration과 easing 값을 통일해 사용자가 테마 전환 시 부드럽고 자연스럽게 느낄 수 있도록 수정했습니다.

이 과정에서 단순 기능 추가보다 “사용자가 느끼는 흐름의 자연스러움”을 더 중요하게 고려했습니다.

---

# 6. 커스텀 UI 적용

초기에는 브라우저 기본 alert/confirm UI를 사용했습니다.

하지만 기본 브라우저 UI는 디자인 일관성이 부족하고, 모바일 환경에서 사용자 경험이 끊기는 느낌이 강하다고 판단했습니다.

그래서 토스 스타일의 커스텀 Toast 및 확인 모달 UI를 직접 구현했습니다.


* 애니메이션 타이밍
* 모바일 화면 대응
* UI 흐름 자연스러움

위 과정을 직접 수정하며 사용자 경험을 개선했습니다.

---

# 7. 느낀 점

AI는 빠르게 초기 구조를 만들고 여러 선택지를 탐색하는 데 매우 유용했습니다.

특히 반복적인 UI 코드 작성이나 초기 기능 구현 속도를 크게 줄여주었고, 다양한 구현 방식을 비교하는 데 도움을 받을 수 있었습니다.

하지만 실제 프로젝트에서는

* 유지보수성
* 사용자 경험
* 데이터 무결성
* 구조 확장성
* 성능

같은 요소를 직접 검증하고 판단해야 했습니다.

이 경험을 통해 AI는 단순 자동 생성 도구라기보다 개발자의 판단을 보조하는 협업 도구에 가깝다고 느끼게 되었습니다.
