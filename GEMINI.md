# Project Rules

## 프로젝트 개요

영어 단어장 웹 앱. 단어와 뜻을 추가/삭제하고, 랜덤 퀴즈로 암기를 도와주는 서비스.

## 기술 스택

- HTML / CSS / Vanilla JavaScript (프레임워크 없이)
- 데이터 저장: localStorage (서버 없이 브라우저에 저장)
- 파일 구조: index.html 하나로 시작

## 일반 원칙

- 모든 코드는 일관된 스타일을 유지해야 합니다.
- 변경 사항은 항상 테스트를 통해 검증되어야 합니다.
- 보안 관련 설정이나 비밀키는 절대 코드에 포함하지 않습니다.

1. 언어 : 모든 대화와 주석, 문서는 한글로 작성한다.
2. 작업 절차 : 모든 답변은 아래 순서를 따른다.
   - 사용자 가치 명세: 이 기능을 왜 만드는지 1~2문장으로 요약.
   - 해결책 개요 : 어떻게 구현할 지 간략한 설명.
   - 코드 : 실제 구현 코드.
   - 실행 방법 : 터미널에서 입력해야 할 명령어.

3. 개발 원칙 : 코드는 간결하게 작성하고, 특히 윈도우 환경에서 오류가 나지 않도록 주의한다.
4. 커밋 규칙
   - 모든 기능 구현이 끝나고 정상 작동을 확인하면 커밋을 제안할 것.
   - 커밋 메세지는 Conventional Commits 규격을 따를 것
     (예: feat: ,fix: ,docs: ,refactor: ,chore: ,revert: 등)
